# 要件定義書

## はじめに

ToMoreBeyond（トモアビヨンド）は、「埋もれた記録を、続く面白さへ」をコンセプトに、未活用資源を再発見し、デジタルとリアルを融合させた体験を提供する企業です。本要件定義書では、ToMoreBeyondのコーポレートサイトの構築に必要な機能要件を、EARS形式で明確に定義します。

サイトは、企業のビジョン・提供価値・プロダクトライン・チーム構成を効果的に伝え、訪問者が企業理念に共感し、製品やサービスに興味を持つことを目的とします。研究所のような温かみとテクノロジー企業の透明感を両立させたデザインにより、ユーザーに「手の感触」と「足跡」を感じさせる体験を提供します。

## 要件

### 要件1: サイト概要とビジョン表示
**目的:** 訪問者として、ToMoreBeyondの企業理念とコンセプトを理解し、企業の独自性と価値を認識したい

#### 受入基準

1. WHEN 訪問者がトップページにアクセスする THEN ToMoreBeyondサイトは、企業名「ToMoreBeyond（トモアビヨンド）」を視覚的に目立つ形で表示する
2. WHEN トップページが表示される THEN ToMoreBeyondサイトは、キャッチコピー「Making hidden traces a lasting wonder（埋もれた記録を、続く面白さへ）」を英語と日本語の両方で表示する
3. WHEN 訪問者が企業コンセプトセクションを閲覧する THEN ToMoreBeyondサイトは、「研究所のような温かみとテクノロジー企業の透明感」という企業の独自性を表現したテキストを表示する
4. WHEN コンセプトセクションが表示される THEN ToMoreBeyondサイトは、「デジタルとリアルを横断した日常に続く面白さの実装」という使命を明確に説明する
5. WHEN WHY NOWセクションが表示される THEN ToMoreBeyondサイトは、スマートフォン普及後の体験均質化と地域資源の未活用という課題認識を提示する
6. WHEN WHY NOWセクションが表示される THEN ToMoreBeyondサイトは、「AIには代替できない手の感触や自分の足跡」という価値提案を明示する

### 要件2: 提供価値の明確化
**目的:** 訪問者として、ToMoreBeyondが提供する具体的な価値を理解し、サービスの恩恵を認識したい

#### 受入基準

1. WHEN 訪問者が提供価値セクションにアクセスする THEN ToMoreBeyondサイトは、「未活用資源の活用」「プライバシーの保護」「デジタルとリアルの融合」の3つの主要価値を明確に区別して表示する
2. WHEN 未活用資源の活用セクションが表示される THEN ToMoreBeyondサイトは、「埋もれた地域資源や個人の経験を再発見し、足跡を残していくサービス」という説明を提供する
3. WHEN プライバシー保護セクションが表示される THEN ToMoreBeyondサイトは、「オンデバイス処理による利用者の行動データの安全な記録・活用」という技術的アプローチを説明する
4. WHEN デジタルとリアル融合セクションが表示される THEN ToMoreBeyondサイトは、「アプリやサービスだけでなく拠点・家具などのリアル側も重視」という方針を示す
5. WHEN 各提供価値が表示される THEN ToMoreBeyondサイトは、訪問者が各価値の具体的な恩恵を直感的に理解できるビジュアルまたはアイコンを併用する

### 要件3: プロダクトライン紹介
**目的:** 訪問者として、ToMoreBeyondが提供する各プロダクトの特徴と目的を理解し、興味のあるサービスを見つけたい

#### 受入基準

1. WHEN 訪問者がプロダクトセクションにアクセスする THEN ToMoreBeyondサイトは、TADATAKA、TOI-RUN、Meet in the middleの3つのプロダクトを明確に区別して表示する
2. WHEN TADATAKAプロダクトカードが表示される THEN ToMoreBeyondサイトは、「あなたの歩いた道が、そのまま日記になる」というコンセプトを提示する
3. WHEN TADATAKAの詳細が表示される THEN ToMoreBeyondサイトは、「GPS・BLE・加速度センサーによる高精度移動記録」「オンデバイス処理による完全プライバシー保護」「美しい地図表示」という3つの主要特徴を列挙する
4. WHEN TADATAKAカードが表示される THEN ToMoreBeyondサイトは、ロードマップへのリンクまたはアクセス方法を提供する
5. WHEN TOI-RUNプロダクトカードが表示される THEN ToMoreBeyondサイトは、「緊急時に最速でトイレを見つけるアプリ（開発中）」という概要を表示する
6. WHEN TOI-RUNカードが表示される THEN ToMoreBeyondサイトは、開発中ステータスを視覚的に明示する（例: "開発中"バッジ）
7. WHEN Meet in the middleプロダクトカードが表示される THEN ToMoreBeyondサイトは、「公平な待ち合わせ場所をAIが提案」という主要機能を説明する
8. WHEN Meet in the middleの詳細が表示される THEN ToMoreBeyondサイトは、「移動コストと時間的公平性を考慮したアルゴリズム」という技術的特徴を説明する
9. WHEN 各プロダクトカードが表示される THEN ToMoreBeyondサイトは、プロダクトごとに視覚的に区別可能なデザイン（色、アイコン、画像など）を適用する
10. IF プロダクトにロードマップが存在する THEN ToMoreBeyondサイトは、該当プロダクトカードにロードマップへのアクセスリンクを表示する

### 要件4: チーム構成の紹介
**目的:** 訪問者として、ToMoreBeyondを構成するチームメンバーの役割と個性を理解し、企業への信頼感を得たい

#### 受入基準

1. WHEN 訪問者がチームセクションにアクセスする THEN ToMoreBeyondサイトは、「多様な視点で面白さを育てる」というチームの価値観を説明するテキストを表示する
2. WHEN チームメンバーリストが表示される THEN ToMoreBeyondサイトは、CDO（山田）、CEO（正留）、CAO（安藤）の3名のメンバー情報を提示する
3. WHEN 各メンバーカードが表示される THEN ToMoreBeyondサイトは、役職名（英語・日本語）、名前（漢字・ローマ字）、担当・メッセージを含める
4. WHEN CDO（山田）のカードが表示される THEN ToMoreBeyondサイトは、「チーフデザイナー。異なる視点で世界を観察し、形にしていく」というメッセージを表示する
5. WHEN CEO（正留）のカードが表示される THEN ToMoreBeyondサイトは、「面白いと思うことだけをする。技術と想像力で未来を創る」という思想を表示する
6. WHEN CAO（安藤）のカードが表示される THEN ToMoreBeyondサイトは、「お金関連担当。俯瞰的な視点でチームを支え、持続可能な仕組みを設計」という役割を表示する
7. WHEN チームメンバーカードが表示される THEN ToMoreBeyondサイトは、各メンバーを視覚的に区別できるレイアウト（写真、アイコン、カラー等）を適用する
8. WHEN チームセクションが表示される THEN ToMoreBeyondサイトは、チームの協働性と多様性を感じさせるビジュアルデザインを提供する

### 要件5: お問い合わせ機能
**目的:** 訪問者として、ToMoreBeyondの製品や会社に興味を持った際に、簡単に連絡できる手段を見つけたい

#### 受入基準

1. WHEN 訪問者がコンタクトセクション（"Let's talk"）にアクセスする THEN ToMoreBeyondサイトは、お問い合わせの目的や方法を明確に説明するテキストを表示する
2. WHEN コンタクトフォームが表示される THEN ToMoreBeyondサイトは、名前、メールアドレス、メッセージ内容を入力できるフィールドを提供する
3. WHEN 訪問者が必須フィールドを空欄のまま送信ボタンをクリックする THEN ToMoreBeyondサイトは、エラーメッセージを表示し、必須項目の入力を促す
4. WHEN 訪問者が無効な形式のメールアドレスを入力する THEN ToMoreBeyondサイトは、「有効なメールアドレスを入力してください」というバリデーションメッセージを表示する
5. WHEN 訪問者が正しい形式ですべての必須フィールドを入力して送信ボタンをクリックする THEN ToMoreBeyondサイトは、フォームデータを指定されたエンドポイントまたはメールアドレスに送信する
6. WHEN フォーム送信が成功する THEN ToMoreBeyondサイトは、「お問い合わせありがとうございます。担当者から連絡いたします」という確認メッセージを表示する
7. WHEN フォーム送信が失敗する THEN ToMoreBeyondサイトは、「送信に失敗しました。しばらくしてから再度お試しください」というエラーメッセージを表示する
8. IF 代替の連絡手段（メールアドレス、SNSリンク等）が存在する THEN ToMoreBeyondサイトは、コンタクトセクションに代替連絡方法を併記する

### 要件6: レスポンシブデザインとアクセシビリティ
**目的:** 訪問者として、どのデバイスからアクセスしても快適にサイトを閲覧し、情報を取得したい

#### 受入基準

1. WHEN 訪問者がデスクトップブラウザからアクセスする THEN ToMoreBeyondサイトは、デスクトップに最適化されたレイアウトで全コンテンツを表示する
2. WHEN 訪問者がタブレットデバイスからアクセスする THEN ToMoreBeyondサイトは、タブレット画面サイズに適応したレイアウトで全コンテンツを表示する
3. WHEN 訪問者がスマートフォンからアクセスする THEN ToMoreBeyondサイトは、モバイル画面サイズに最適化されたレイアウトで全コンテンツを表示する
4. WHEN モバイル表示時にナビゲーションメニューが必要になる THEN ToMoreBeyondサイトは、ハンバーガーメニューまたは同等のモバイルナビゲーションUIを提供する
5. WHEN 訪問者が画面サイズを変更する（ブラウザウィンドウのリサイズ） THEN ToMoreBeyondサイトは、リアルタイムでレイアウトを調整し、コンテンツを適切に表示する
6. WHEN 訪問者がキーボードのみで操作する THEN ToMoreBeyondサイトは、すべてのインタラクティブ要素（リンク、ボタン、フォーム等）にキーボードでアクセス可能にする
7. WHEN スクリーンリーダーが使用される THEN ToMoreBeyondサイトは、画像に適切なalt属性、見出しに階層構造、リンクに明確なテキストを提供する
8. WHEN 訪問者がダークモードを有効にしている THEN ToMoreBeyondサイトは、システム設定を検出してダークモードに対応した配色を適用する（任意）

### 要件7: パフォーマンスとSEO
**目的:** 訪問者として、サイトの高速な読み込みと検索エンジンでの発見性を期待したい

#### 受入基準

1. WHEN 訪問者が初回アクセスする THEN ToMoreBeyondサイトは、First Contentful Paint（FCP）を2.5秒以内に達成する
2. WHEN ページが読み込まれる THEN ToMoreBeyondサイトは、Largest Contentful Paint（LCP）を3秒以内に達成する
3. WHEN 訪問者がページ内でインタラクションを行う THEN ToMoreBeyondサイトは、First Input Delay（FID）を100ミリ秒以内に抑える
4. WHEN 画像がページに読み込まれる THEN ToMoreBeyondサイトは、遅延読み込み（lazy loading）を適用して初期読み込み時間を短縮する
5. WHEN サイトが検索エンジンにインデックスされる THEN ToMoreBeyondサイトは、各ページに適切なメタタイトル、メタディスクリプション、OGPタグを含める
6. WHEN 検索エンジンクローラーがアクセスする THEN ToMoreBeyondサイトは、構造化データ（JSON-LD）を提供して企業情報・プロダクト情報を明示する
7. WHEN サイトがSNSでシェアされる THEN ToMoreBeyondサイトは、適切なOGP画像、タイトル、説明文を表示してシェアプレビューを最適化する
8. WHEN sitemap.xmlが要求される THEN ToMoreBeyondサイトは、すべての主要ページを含むXMLサイトマップを提供する

### 要件8: デザインシステムとブランディング
**目的:** 訪問者として、ToMoreBeyondのブランドアイデンティティを一貫して感じ、企業の世界観に浸りたい

#### 受入基準

1. WHEN サイト全体が表示される THEN ToMoreBeyondサイトは、「研究所のような温かみとテクノロジー企業の透明感」を表現する統一されたデザインシステムを適用する
2. WHEN カラーパレットが使用される THEN ToMoreBeyondサイトは、ブランドカラーを一貫して使用し、セクションごとに視覚的な調和を保つ
3. WHEN タイポグラフィが表示される THEN ToMoreBeyondサイトは、可読性と視覚的階層を両立したフォントファミリーとサイズを適用する
4. WHEN 見出しやセクションタイトルが表示される THEN ToMoreBeyondサイトは、親しみやすさと先進性を感じさせるタイポグラフィスタイルを使用する
5. WHEN ビジュアル要素（アイコン、イラスト、写真等）が表示される THEN ToMoreBeyondサイトは、「手の感触」と「足跡」を連想させるビジュアルモチーフを組み込む
6. WHEN アニメーションやトランジションが使用される THEN ToMoreBeyondサイトは、過度に派手でなく、ユーザー体験を向上させる適度なアニメーションを適用する
7. WHEN ボタンやCTA（Call to Action）が表示される THEN ToMoreBeyondサイトは、視覚的に目立ち、クリック可能であることが明確なデザインを適用する
8. WHEN サイト全体のレイアウトが構築される THEN ToMoreBeyondサイトは、余白（ホワイトスペース）を効果的に使用し、情報の見やすさと呼吸感を確保する

### 要件9: ナビゲーションとユーザーフロー
**目的:** 訪問者として、サイト内をスムーズに移動し、求める情報に素早くアクセスしたい

#### 受入基準

1. WHEN トップページが表示される THEN ToMoreBeyondサイトは、ヘッダーにグローバルナビゲーションメニューを配置する
2. WHEN グローバルナビゲーションメニューが表示される THEN ToMoreBeyondサイトは、主要セクション（概要、提供価値、プロダクト、チーム、お問い合わせ）へのリンクを提供する
3. WHEN 訪問者がナビゲーションリンクをクリックする THEN ToMoreBeyondサイトは、対象セクションへスムーズスクロールまたはページ遷移を実行する
4. WHEN 訪問者がページをスクロールする THEN ToMoreBeyondサイトは、現在閲覧中のセクションに対応するナビゲーション項目をハイライト表示する（任意）
5. WHEN 訪問者がフッターにアクセスする THEN ToMoreBeyondサイトは、企業情報、プライバシーポリシー、利用規約、SNSリンク等の補足情報を提供する
6. WHEN 訪問者がロゴをクリックする THEN ToMoreBeyondサイトは、トップページまたはトップセクションに戻る
7. IF サイトが複数ページ構成である THEN ToMoreBeyondサイトは、パンくずリスト（Breadcrumb）を提供して現在位置を明示する
8. WHEN 訪問者が404エラーページにアクセスする THEN ToMoreBeyondサイトは、わかりやすいエラーメッセージとトップページへの誘導リンクを表示する

### 要件10: ロードマップとCMS連携（拡張機能）
**目的:** 運営者として、プロダクトのロードマップや企業情報を容易に更新し、訪問者に最新情報を提供したい

#### 受入基準

1. WHEN 運営者がCMS（コンテンツ管理システム）にログインする THEN ToMoreBeyondサイトは、プロダクト情報、チーム情報、お問い合わせ設定を編集可能なインターフェースを提供する
2. WHEN 運営者がプロダクト情報を更新する THEN ToMoreBeyondサイトは、変更内容をリアルタイムまたは即座に反映する
3. WHEN 運営者がロードマップ情報を追加・編集する THEN ToMoreBeyondサイトは、各プロダクトに関連するロードマップデータを保存し、フロントエンドに表示する
4. WHEN 訪問者がロードマップセクションまたはページにアクセスする THEN ToMoreBeyondサイトは、各プロダクトの開発ステージ、予定機能、リリース時期を視覚的に表示する
5. WHEN ロードマップが更新される THEN ToMoreBeyondサイトは、最新情報を訪問者に通知する仕組み（例: "NEW"バッジ、更新日表示）を提供する
6. IF CMSが導入されていない THEN ToMoreBeyondサイトは、静的ファイル（JSON、Markdown等）でコンテンツを管理し、ビルド時に反映する
7. WHEN 運営者がチーム情報を更新する THEN ToMoreBeyondサイトは、新メンバーの追加や既存メンバー情報の変更を即座にサイトに反映する
8. WHEN 運営者がブログ記事や企業ニュースを投稿する THEN ToMoreBeyondサイトは、新着情報セクションまたはブログページに記事を表示する（任意拡張機能）
